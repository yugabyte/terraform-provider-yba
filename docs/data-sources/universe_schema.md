# yba_universe_schema (Data Source)

Discover the schema (namespaces and tables) of a universe. Use this data source to find database/keyspace names and table UUIDs for backups.

This is particularly useful for dynamically backing up all databases without hardcoding
their names, or for locating specific table UUIDs for table-level backups.

## Example Usage

```terraform
// List all user-created namespaces (databases and keyspaces) in a universe.
data "yba_universe_schema" "namespaces" {
  universe_uuid = "<universe-uuid>"
}

// Include tables and filter to YSQL only, for table-level backup configuration.
data "yba_universe_schema" "ysql_tables" {
  universe_uuid             = "<universe-uuid>"
  table_type                = "PGSQL_TABLE_TYPE"
  include_tables            = true
  include_system_namespaces = false
}
```

## Schema

### Required

- `universe_uuid` (String) The UUID of the universe to inspect.

### Optional

- `include_system_namespaces` (Boolean) Include system namespaces in the results. Default: false.
- `include_tables` (Boolean) Include tables in the results. Default: false. Enable this only when you need table UUIDs for table-level backups.
- `table_type` (String) Filter by table type. Allowed values: YQL_TABLE_TYPE (YCQL), PGSQL_TABLE_TYPE (YSQL). If not specified, returns all types.

### Read-Only

- `id` (String) The ID of this resource.
- `namespace_names` (List of String) List of namespace names. Convenient for use with backup keyspaces argument.
- `namespaces` (List of Object) List of namespaces (databases/keyspaces) in the universe. (see [below for nested schema](#nestedatt--namespaces))
- `tables` (List of Object) List of tables in the universe. Only populated when include_tables is true. (see [below for nested schema](#nestedatt--tables))
- `ycql_keyspace_names` (List of String) List of YCQL keyspace names only.
- `ysql_database_names` (List of String) List of YSQL database names only.
