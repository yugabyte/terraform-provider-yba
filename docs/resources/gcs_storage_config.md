# yba_gcs_storage_config (Resource)

GCS (Google Cloud Storage) Configuration for YugabyteDB Anywhere backups.

The `yba_backup`, `yba_backup_schedule`, and `yba_restore` resources reference storage
configurations via the `storage_config_uuid` attribute.

For more details, see the [YugabyteDB Anywhere Configure Storage documentation](https://docs.yugabyte.com/stable/yugabyte-platform/back-up-restore-universes/configure-backup-storage/).

## Example Usage

```terraform
// GCS storage configuration with service account credentials.
resource "yba_gcs_storage_config" "gcs" {
  name            = "my-gcs-config"
  backup_location = "gs://my-bucket/yugabyte-backups"
  credentials     = file("~/.gcp/service-account.json")
}

// GCS storage configuration using GKE workload identity.
resource "yba_gcs_storage_config" "gcs_iam" {
  name            = "gcs-workload-identity"
  backup_location = "gs://my-bucket/yugabyte-backups"
  use_gcp_iam     = true
}
```

## Migration from `yba_storage_config_resource`

If you previously managed GCS storage configurations with the deprecated [`yba_storage_config_resource`](storage_config_resource), switch the block type, lift `gcs_credentials.application_credentials` to the top-level `credentials` field, rename `config_name` to `name`, and re-import the state:

```hcl
# Before (deprecated)
resource "yba_storage_config_resource" "gcs" {
  name            = "GCS"
  config_name     = "my-gcs-config"
  backup_location = "gs://my-bucket/yugabyte-backups"
  gcs_credentials {
    application_credentials = file("~/.gcp/service-account.json")
  }
}

# After
resource "yba_gcs_storage_config" "gcs" {
  name            = "my-gcs-config"
  backup_location = "gs://my-bucket/yugabyte-backups"
  credentials     = file("~/.gcp/service-account.json")
}
```

Then re-bind the state to the new address:

```sh
terraform state rm yba_storage_config_resource.gcs
terraform import yba_gcs_storage_config.gcs <config-uuid>
```
